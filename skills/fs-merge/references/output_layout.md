# 출력 xlsx 레이아웃

단일 시트 `FS`에 PL → MS → BS 순으로 3개 섹션이 들어간다.

## 컬럼 배치 (모든 섹션 공통)

| 컬럼 | 용도 |
|------|------|
| A | 섹션 마커 (` - `) |
| B | 섹션명 / 구분 / 계정과목 |
| C | 실계정 (0=소계, 1=leaf, blank=섹션헤더) |
| D | Group (BS만 사용. PL/MS는 비움. 사용자가 수기로 분류 작업할 자리) |
| E~ | 연도 컬럼 (오래된 것부터 최신 순. 헤더는 결산기말일 date) |
| 연도 뒤 3개 컬럼 | `재계산_차이`, `OCR_의심`, `매칭_의심` 플래그 |

## 섹션 구조

각 섹션은 다음 블록으로 구성:

```text
A: " - "    B: "PL"                      <- 섹션 마커 행 (한 행)
            (빈 행)                        <- 시각적 분리
            B: "구분"  C: "실계정"  E~: 날짜  뒤: 플래그명   <- 헤더 행
            (period_notes 행, 있을 때만)   <- 12M 아닌 기간 라벨 표시
            데이터 행 (계정과목별)
            (빈 행)
            B: "CHECK"                    <- 검증 섹션 시작
            소계 검증 행들
            (빈 행)
```

마지막 BS 섹션 뒤에 `Ref` 섹션을 둘 수 있음 (선택). `당기제품제조원가` 같은 statement-간 일치 검증용.

## 데이터 셀 포맷

- 숫자 셀: `#,##0;(#,##0);-` (음수는 괄호, 0은 대시)
- 날짜 헤더: `yyyy-mm-dd`
- 헤더 행 / 섹션 마커 / CHECK·Ref 라벨: bold
- period_notes 행: italic, size 9, center alignment

## 차감계정 부호 규약

차감계정(contra-account)은 **반드시 음수로 입력**:

- `감가상각누계액` (유형자산 차감)
- `대손충당금` (매출채권 차감)
- `재고자산평가충당금`
- `사채할인발행차금`
- 등

이렇게 해야 ②유형자산 같은 소계의 sum check (`=Sub - SUM(children)`)가 자동으로 동작한다. 표시 포맷 `#,##0;(#,##0);-` 가 음수를 괄호로 보여주므로 시각적으로는 원본 BS와 동일하게 보인다.

위배 시 발생하는 문제: `유형자산 = 토지 + 건물 + 차량 + 감누 + 비품 + 감누 + ...` 식 sum check가 두 배 차이(잔여 = 감누×2)로 항상 fail.

## 섹션 헤더 행

`[자산]`, `[부채]`, `[자본]` 같은 시각적 구분 행은:

- `is_leaf: false`
- `is_section_header: true` ← 이걸로 구분
- 모든 `values: null`
- CHECK 대상 아님 (checks 배열에 포함하지 말 것)

`is_section_header: true` 가 있으면 build script는 실계정(C) 컬럼을 비워둔다 (소계의 0과 구분). 시각적으로 섹션을 가르는 라벨 역할.

## 비12개월 기간 표시 (period_notes)

PL/MS의 경우 12개월이 아닌 기간(반기·분기·월별)은 헤더에 명시:

```json
"fiscal_year_ends": ["2022-12-31", "2023-12-31", "2024-12-31", "2025-06-30"],
"period_notes":     [null,         null,         null,         "6M interim"]
```

build script는 날짜 헤더 행 바로 아래에 라벨을 italic으로 한 줄 추가한다. BS 컬럼에는 라벨이 무의미(잔액은 시점 개념)지만 PL/MS와 컬럼이 공유되므로 그대로 표시되어도 사용자가 헷갈리지 않는다.

라벨 예시: `"6M interim"`, `"3M Q1"`, `"제출용 가결산"`, etc.

## CHECK 행 — 두 가지 형식

### 형식 1: 단순 합 (legacy)

자식 행들이 모두 양수 합인 경우:

```json
{"label": "Ⅳ.판매관리비 = Σ leaves",
 "subtotal_idx": 8,
 "child_indices": [9, 10, 11, ..., 29]}
```

생성 수식:
- 자식 행이 인접하면 `=E{소계행} - SUM(E{첫}:E{끝})`
- 비인접하면 `=E{소계행} - (E{a}+E{b}+E{c})`

### 형식 2: 부호 있는 선형 결합 (terms)

매출총이익 = 매출 - 매출원가 같은 비가산 관계:

```json
{"label": "Ⅲ.매출총이익 = Ⅰ.매출액 - Ⅱ.매출원가",
 "subtotal_idx": 7,
 "terms": [{"idx": 0, "sign": 1},
           {"idx": 2, "sign": -1}]}
```

생성 수식:
```
=E{매출총이익} - (E{매출액} - E{매출원가})
```

부호는 `1` 또는 `-1`. 더 복잡한 가중치는 지원 안 함 (회계 검증식엔 ±1만 등장).

### 표준 PL 비가산 체크리스트

해당 계정이 master에 존재하면 항상 포함:

| 검증식 | terms |
|--------|-------|
| `상품매출원가 = 기초재고 + 당기매입 - 기말재고` | (기초, +1), (매입, +1), (기말, -1) |
| `Ⅲ.매출총이익 = Ⅰ.매출액 - Ⅱ.매출원가` | (매출, +1), (원가, -1) |
| `Ⅴ.영업이익 = Ⅲ.매출총이익 - Ⅳ.판관비` | (매총, +1), (판관비, -1) |
| `Ⅷ.소득세차감전이익 = Ⅴ.영업이익 + Ⅵ.영업외수익 - Ⅶ.영업외비용` | (영업이익, +1), (영외수익, +1), (영외비용, -1) |
| `Ⅹ.당기순이익 = Ⅷ.소득세차감전이익 - Ⅸ.소득세등` | (세전, +1), (세금, -1) |

### 표준 BS 합산 체크리스트

| 검증식 | 형식 |
|--------|------|
| `①당좌자산 = Σ leaves` | child_indices |
| `②재고자산 = Σ leaves` | child_indices |
| `Ⅰ.유동자산 = ①당좌 + ②재고` | child_indices (소계의 합) |
| `①투자자산 = Σ leaves` | child_indices |
| `②유형자산 = Σ leaves` (감누 음수 포함) | child_indices |
| `Ⅱ.비유동자산 = ①+②+③+④` | child_indices |
| `자산총계 = Ⅰ + Ⅱ` | child_indices |
| `Ⅰ.유동부채 = Σ leaves` | child_indices |
| `Ⅱ.비유동부채 = Σ leaves` | child_indices |
| `부채총계 = Ⅰ + Ⅱ` | child_indices |
| `자본총계 = Σ 자본 소계` | child_indices |
| `부채와자본총계 = 부채총계 + 자본총계` | child_indices |

자본 롤포워드 (`기말RE = 시초RE + 당기순이익`) 는 자본금/이익잉여금이 분리된 경우만 추가. 소규모법인 단일 자본금 행이면 skip.

## 연도 컬럼 결정 규칙

`fiscal_year_ends` 배열을 오래된 것부터 최신 순으로 정렬 후 E부터 순차 배치.

**N개의 FS 파일이 있을 때**:
- PL/MS는 일반적으로 첫 결산년도(가장 오래된 FS의 prior year)는 없음 → 해당 셀 비움
- BS는 보통 N개 파일에서 N+1개 시점의 잔액 (또는 최소 N개)
- 누락된 셀은 `null` 로 표기 → xlsx에서 빈 셀로 처리됨
