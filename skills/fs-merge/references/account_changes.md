# 계정과목 변경 처리 가이드

연도별 chart of accounts 변경(rename / split / merge)에 일관된 원칙으로 대응하기 위한 결정 트리.

## 세 가지 변경 패턴

### Pattern A — 단순 리네임 (1 → 1)

같은 경제적 실질의 계정이 이름만 바뀜.

예시:
- `매입채무` → `외상매입금` (단순 표기 변경)
- `급여` → `직원급여`
- `잡급` → `임시직급여`

### Pattern B — 분리 (1 → N)

하나의 계정이 여러 개로 세분화됨.

예시:
- `현금및현금성자산` → `현금` + `보통예금` + `당좌예금`
- `유형자산` 단일 표시 → `토지` + `건물` + `차량` + ... 개별 표시

### Pattern C — 통합 (N → 1)

여러 계정이 하나로 합쳐짐. (실무에서 흔하지 않음)

예시:
- `소모품비` + `사무용품비` → `소모품비`로 통합

## 결정 트리

```text
master에 없는 계정이 older에서 발견됨
│
├─ 위치(BS/PL 내 구간)가 동일한가?
│  │
│  ├─ Yes — 인접 sum check 가 양쪽 모두 잘 맞는가?
│  │  │
│  │  ├─ Yes (sum 일치) → Pattern A 의심: rename
│  │  │  ├─ 액션: master 이름으로 행 통합. older 값을 master 행에 채움
│  │  │  └─ 플래그: 매칭_의심 = "{older_name} → {master_name} 리네임"
│  │  │
│  │  └─ No (sum 안 맞음) → Pattern B 의심: 분리되었음
│  │     ├─ 액션: older의 통합 계정을 별도 행으로 보존 (master 행에 합치지 말 것)
│  │     ├─ master의 분리된 행들은 older 연도에 빈 셀
│  │     └─ 양쪽 모두 매칭_의심 플래그
│  │
│  └─ No (위치 다름) → 단발성 또는 신규/소멸 계정
│     ├─ 액션: 별도 행으로 신규 삽입 (older만 값 채움)
│     └─ 플래그: 매칭_의심 = "master에 없는 계정 — {연도} 단발성"
│
└─ master 행이 older에 없음 (반대 케이스)
   ├─ 액션: master 행 그대로 두고 older 연도 셀만 비움
   └─ 플래그 불필요 (신규 계정은 자연스러움)
```

## 케이스 스터디

### 매입채무 → 외상매입금 (Pattern A — rename)

조건 확인:
- 두 계정 모두 `유동부채` 첫 번째 leaf 위치 ✓
- 2023 BS: 유동부채 36,457,692 = 매입채무 + 예수금 + 부가세예수금 + 미지급비용 = 잘 맞음 ✓
- 2025/06 BS: 유동부채 34,623,150 = 외상매입금 + 예수금 + 부가세예수금 + 미지급비용 = 잘 맞음 ✓

→ Pattern A. master(외상매입금) 행에 2023 매입채무 값을 채우고 플래그.

### 현금및현금성자산 → 현금 + 보통예금 (Pattern B — split)

조건 확인:
- 위치(당좌자산 첫 leaf 들) 동일 ✓
- 2023 BS: 당좌자산 75,062,942 = 현금및현금성자산 75,062,942 (단일 leaf)
- 2025/06 BS: 당좌자산 69,732,765 = 현금 222,214 + 보통예금 68,754,741 + 미수금 755,810 (3개 leaves)

→ Pattern B. 통합 계정을 별도 행으로 보존.
- master에 현금/보통예금/미수금 행 그대로 (older 연도 빈 셀)
- 별도 추가 행: 현금및현금성자산 (older 연도만 값, master 연도 빈 셀)
- 양쪽에 매칭_의심 플래그

### 잡급 (단발성)

조건 확인:
- 2023 PL에만 333,870. 2022·2024·2025-H1 모두 없음.
- master(2025-H1)에 잡급 없음.
- 위치: 직원급여 바로 아래 (older 한 곳에서만 등장)

→ 결정 트리의 "위치 다름 / 단발성" 분기.
- 별도 행으로 master의 직원급여 바로 아래에 삽입
- 2023 컬럼만 값, 나머지 연도는 빈 셀
- 플래그: "master에 없는 계정 — 2023년 단발성"

## 절대 하지 말 것

- **String similarity 매칭** (e.g., 매입채무 vs 외상매입금이 비슷하니까 같다고 판단) — 한국어 회계 용어는 비슷한 단어가 다른 의미인 경우가 많음 (`미수금` vs `미수수익`, `미지급금` vs `미지급비용`)
- **계정 잔액 크기 비교만으로 매칭** (10M vs 11M이니까 같은 계정) — 우연 일치 가능성
- **임의 통합** — 사용자가 명시 허락하지 않은 한 별도 행 보존이 default
- **플래그 없이 통합** — 모든 변경은 매칭_의심 플래그로 기록

## 의심스러우면

별도 행 보존 + 매칭_의심 플래그가 항상 안전. 사용자(KICPA)가 보고 직접 통합 여부 판단함.
